zkpay docs

Stablecoin checkout infrastructure for Sui apps.

zkpay starts with one developer loop: create a payment intent, send the payer to hosted checkout, route gas clearly, and verify the receipt before fulfillment.

Quickstart

The first public code path is intentionally narrow. It gives builders a deterministic payment object before adding Sui RPC, zkLogin proving, webhook delivery, and merchant operations. 

npm install
npm run check
npm run cli:demo
npm run dev

Payment model

A zkpay payment carries amount, coin, receiver, label, metadata, nonce, timestamps, checkout URL, payment URI, gas route decision, and receipt verification result. 

PaymentIntent
-> hosted checkout
-> GasRouteDecision
-> PaymentReceipt
-> verification before fulfillment

CLI

The CLI creates payment links and JSON payloads for scripts, demos, and merchant backend prototyping. 

zkpay link create \
  --amount 20 \
  --coin USDC \
  --receiver 0x84f \
  --label "API credits" \
  --json
