zkpay docs

Stablecoin checkout infrastructure for Sui apps.

zkpay turns payment links, zkLogin checkout, gas routing, and verifiable receipts into one integration surface for Sui builders. This docs site will track the product API as implementation begins.

Quickstart

The first developer loop is intentionally small: create a payment intent, send the payer to checkout, then verify the receipt before fulfillment. 

zkpay link create
  --amount 20
  --coin USDC
  --receiver 0x...
  --label "API credits"

Payment model

A zkpay payment should carry amount, coin type, receiver, expiry, nonce, and merchant metadata. Checkout builds the Sui transaction and asks the payer to authorize the exact payment.

Gas policy

zkpay should use Sui gasless stablecoin transfers when the route is eligible. Checkout logic that needs extra programmable transaction work can use sponsored gas. Unsupported paths should return an explicit fallback reason.

Receipt verification

Merchant systems should verify digest, receiver, amount, coin type, nonce, and receipt status before marking an order paid. 

const receipt = await zkpay.verifyPayment("zkp_123");

if (receipt.status === "succeeded") {
  await fulfillOrder(receipt.orderId);
}
