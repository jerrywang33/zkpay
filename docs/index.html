Quickstart

The first public package is intentionally narrow. It gives builders a deterministic payment object, hosted wallet settlement, Sui RPC verification, optional onchain receipt binding, and a CLI/API surface for merchant prototypes. 

npm install zkpay-sh@next
zkpay link create --amount 20 --coin USDC --receiver 0x84f --json

Install

The public alpha package bundles the SDK, core primitives, and CLI. Use the @next tag while the package is still in alpha. Node 22 or newer is recommended because the Sui TypeScript SDK is now part of the package surface. 

npm install zkpay-sh@next
npm install -g zkpay-sh@next
import { ZkpayClient } from "zkpay-sh";

const zkpay = new ZkpayClient();
const payment = zkpay.createPayment({
  amount: "20",
  coin: "USDC",
  receiver: "0x84f",
  label: "API credits"
}, {
  checkout: {
    network: "testnet",
    coinType: "0x...::usdc::USDC",
    decimals: 6,
    bindingPackageId: "0x..."
  }
});

Testnet settlement

v0.2 adds a Sui testnet loop: build a wallet-submittable transaction, open hosted checkout, get the digest from the wallet, then verify the digest against Sui RPC before fulfillment. 

const built = zkpay.buildSuiPaymentTransaction({
  intent: payment.intent,
  payer: "0x...",
  coinType: "0x...::usdc::USDC",
  decimals: 6,
  binding: {
    packageId: "0x..."
  }
});

const result = await zkpay.verifySuiPayment({
  intent: payment.intent,
  txDigest: "H2j...",
  coinType: "0x...::usdc::USDC",
  expectedSender: "0x...",
  binding: {
    packageId: "0x..."
  }
});
https://zkpay.sh/pay/zkp_...?intent=...&network=testnet&coinType=0x...::usdc::USDC&decimals=6

// hosted checkout returns txDigest and a /payments/verify/sui payload

Payment model

A zkpay payment carries amount, coin, receiver, label, metadata, nonce, timestamps, checkout URL, payment URI, gas route decision, and receipt verification result. 

PaymentIntent
-> hosted checkout
-> GasRouteDecision
-> PaymentReceipt
-> verification before fulfillment

Webhooks

Webhook events use t=...,v1=... HMAC headers so merchant systems can verify asynchronous payment events before fulfillment, ledger updates, or reconciliation. Managed endpoints can also carry their own signing secret and send a signed test event before production orders depend on them. 

const event = zkpay.createWebhookEvent({
  type: "payment.succeeded",
  paymentId: payment.intent.id,
  intent: payment.intent,
  receipt
});

const signatureHeader = zkpay.signWebhookEvent(event);

CLI

The CLI creates payment links and JSON payloads for scripts, demos, and merchant backend prototyping.

  --amount 20 \
  --coin USDC \
  --receiver 0x84f \
